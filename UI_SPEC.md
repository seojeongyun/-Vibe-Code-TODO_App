# UI Spec

## 1) 메인 화면 (Home)

### 레이아웃
- 화면은 상/하 2분할 구조
- 상단: 월간 캘린더(월 이동 + 날짜 선택)
- 하단: 연간 Heatmap (GitHub 잔디형, 한 연도 단위)

### 상단(헤더)
- 좌상단 타이틀: `TODO`
- 우상단에 설정 버튼 1개만 제공
- 기본 화면에 Timezone 문구는 표시하지 않음

### 상단(월간 캘린더)
- 월 단위 표시(예: 2026-02)
- 좌/우 버튼으로 월 이동
  - 왼쪽: 이전 달
  - 오른쪽: 다음 달
- 날짜 클릭 시 Daily 화면으로 전환(선택 날짜 전달)
- **주의:** 날짜 클릭은 "이동/조회"만 수행하며, 자동으로 TODO Row를 생성하지 않음

### 캘린더 완료 표시
- 해당 날짜에 isDone=true TODO가 1개라도 있으면 캘린더 날짜 셀 배경을 밝은 회색으로 표시
- 달성도(achievement)와 무관한 고정 색상
- 표시 갱신은 즉시 반영되어야 함(새로고침 없이)

## 하단(연간 Heatmap)

### 구성
- Heatmap은 **한 연도(1월~12월) 단위**로 구성된다
  - 한 연도 화면에는 12개의 월별 히트맵 군집만 존재
  - 다른 연도의 월이 섞이면 안 됨
- Heatmap 상단에 **현재 연도(YYYY) 라벨** 표시 + 좌/우 연도 이동 버튼 제공
- 12개월 군집은 **가로 1줄(1D)** 로 나열되고 **좌우 스크롤**로 탐색
  - 월 내부 날짜 셀은 2D(기존 월 그리드) 유지
- 각 월별 군집에 월 라벨(1월~12월) 표시
- 날짜 셀 클릭 시 Daily 화면으로 전환(해당 날짜 선택)

### 히트맵 표현(밀도)
- 셀 크기는 현재 대비 약 2배
- gap은 촘촘하게 유지

### 히트맵 색상 규칙
- 각 날짜 셀 색상은 해당 날짜의 달성도(0~100)에 따라 농도가 달라짐(0~5 단계)
- Heatmap 기본색 제거
  - achievement=null 또는 achievement=0 인 셀은 **투명(배경 그대로)**
    - 필요 시 얇은 테두리만 표시
  - achievement>0 인 경우에만 테마 색 농도(1~5단계) 적용

### 히트맵 테마 선택
- 사용자가 히트맵 색상 테마를 선택 가능
- 최소 3가지 테마 제공
  1. 기본 GitHub Green
  2. Blue 계열
  3. Purple 계열
- 테마 선택 UI는 Heatmap 상단에 작은 드롭다운/버튼 형태로 제공
- 선택한 테마는 localStorage에 저장하여 새로고침 후 유지

### 캘린더 ↔ Heatmap 연도 동기화
- Heatmap 연도 변경 → 캘린더 연도 변경
- 캘린더 연도 변경(12월→1월 등) → Heatmap 연도 변경

### Home 갱신 규칙(중요)
- Daily에서 isDone/percent 변경 또는 저장(add/update/delete)이 발생하면,
  Home의 Heatmap 색과 캘린더 완료 표시(회색)는 **즉시 갱신**되어야 한다(새로고침 없이).

### 배경
- 전체 앱 배경은 하늘색 계열 그라데이션
- 상단 색이 너무 진하지 않게, 위쪽이 더 연하고 부드럽게 시작하도록 조정한다.
  - 예: linear-gradient(to bottom, #f7fdff 0%, #e7f7ff 45%, #f8fbff 100%)
- 텍스트 가독성 유지

### 메인 화면에서 보이지 않아야 하는 것
- TODO 리스트(일간 상세)는 메인 화면에 표시하지 않음
  - TODO 관리는 Daily 화면에서만 수행

## 2) 일간 TODO 화면 (Daily)

### 진입 조건
- 메인 캘린더 날짜 클릭 또는 Heatmap 날짜 셀 클릭으로 진입
- 선택된 날짜(YYYY-MM-DD)가 Daily 화면의 기준 날짜가 됨

### 상단
- 선택한 날짜 표시 (YYYY-MM-DD)
- "달력으로" 또는 뒤로가기 버튼 제공 → 메인 화면으로 복귀

### 본문: TODO 테이블(일정 관리)
- 세로(y축): TODO 항목(행)
- 가로(x축): 시작시간 / 끝시간 / 할 일(활동 내용) / 완료 여부 / 완료 %

#### TODO Row 컬럼(좌→우)
1) 시작시간(startTime): 시간/분 텍스트 입력 (HH/MM)
2) 끝시간(endTime): 시간/분 텍스트 입력 (HH/MM)
3) 할 일(title): 활동 내용 입력(텍스트, 필수)
4) 완료 여부(isDone): 체크박스
5) 완료 %(percent): 0/25/50/75/100 버튼 선택

### 입력/동작 규칙
- startTime/endTime 입력
  - hour/minute는 텍스트 입력(type="text", inputMode="numeric", maxLength=2)
  - 입력 중에는 값이 사라지지 않도록 draft 유지
  - 포커스 아웃 시 범위 자동 보정 후 "HH:MM" 형식으로 저장
  - **추가 UX:** 입력칸 클릭 후 숫자를 입력하면 기존 값이 자동으로 대체되도록 한다
    - 예: 12 → 클릭 → 11 입력 시 12가 지워지고 11로 교체(전체 선택 처리)
- isDone=false이면 percent는 0으로 유지
  - percent 입력은 비활성화 처리
- isDone=true일 때 percent 버튼 선택 가능
- percent는 0/25/50/75/100 중 하나로 저장
  - 기존 값이 다른 경우 가장 가까운 값으로 보정
- 입력/수정/삭제는 즉시 저장(로컬 IndexedDB)
- 새로고침 후에도 데이터 유지

### 테이블 정렬(중요)
- 헤더의 "완료" 텍스트가 완료 체크박스 컬럼과 수직으로 정렬되어야 한다.
  - 헤더/셀 모두 수직 가운데 정렬

### details(상세)
- 각 TODO 행 아래에 textarea로 details 입력 가능
- 즉시 저장
- 기존 데이터에 details 없으면 null로 보정

### 설정 패널
- 헤더의 설정 버튼을 누르면 설정 패널을 연다
- [데이터 백업/복원]
  - Export: JSON 파일 다운로드
  - Import(replace): JSON 파일 업로드 → 확인 후 복원
- [About]
  - 기준 타임존: Asia/Seoul
- [고급]
  - Persisted?/Persist 요청은 고급 섹션에 표시

### 날짜 클릭 시 동작 규칙(중요)
- Daily 화면 진입은 "기존 데이터 로드(getTodosByDate)"가 기본이며,
  날짜 클릭 자체로 새로운 TODO가 자동 생성되면 안 됨
- 새로운 TODO Row는 사용자가 "추가" 버튼을 눌렀을 때만 생성됨
